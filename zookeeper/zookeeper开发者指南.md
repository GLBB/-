# Zookeeper 开发者指南

## 理解 zookeeper 如何工作

### zookeeper 数据模型

zookeeper 有层级命名空间，就像分布式文件系统一样。

和分布式文件系统不同的是：在命名空间下, 每一个节点有数据和子节点。

zookeeper 就像一个文件系统，允许一个命名空间即是文件，又是目录。

到每一个节点的路径是规范的，必须是**绝对路径**，以斜线分割。

这个路径采用 Unicode 编码，但是下面编码是不允许使用的。

* null 字符 \u0000
* \u0001 - \u007F
* \u009F
* \ud800 - \uF8FF,  \uFFF0 - \u FFFF

字符 **.** 是允许成为路径的一部分的，但是不允许单独使用。（私人建议：尽量不要使用）

其中 “zookeeper” 是保留字。
